# V2V Eval Phase 2 (1.3B) - Standalone Package

Self-contained evaluation pipeline for OmniAvatar 1.3B V2V phase 2 checkpoints.

## Directory layout

```
omniavatar_1.3B_inference/
├── scripts/
│   ├── run_v2v_eval_phase2_1.3B.sh   # Main eval script
│   ├── generate_input_files.sh        # Generate input txt files from validation_data/
│   ├── inference_v2v.py               # V2V inference
│   └── eval_aligned_crops.py          # GT-aligned crop metrics
├── configs/
│   └── inference_v2v_1.3B.yaml        # 1.3B inference config
├── OmniAvatar/                        # Core inference package
├── pretrained_models/                 # ← you provide these
│   ├── Wan2.1-T2V-1.3B/
│   │   ├── diffusion_pytorch_model.safetensors
│   │   ├── models_t5_umt5-xxl-enc-bf16.pth
│   │   └── Wan2.1_VAE.pth
│   └── wav2vec2-base-960h/
├── checkpoints/                       # ← you provide these
│   ├── config.json                    # Training config (in_dim=65, etc.)
│   ├── step-2500.pt                   # Trained checkpoint(s)
│   └── auxiliary/                     # InsightFace, SyncNet, etc.
│       ├── models/buffalo_l/
│       ├── sfd_face.pth
│       ├── syncnet_v2.model
│       └── i3d_torchscript.pt
├── validation_data/                   # ← you provide these
│   ├── hdtf/
│   │   ├── <sample_name>/sub_clip.mp4, audio.wav, prompt.txt
│   │   └── face_cache/<sample_name>_face_cache.pt
│   └── hallo3/
│       └── (same structure)
├── latentsync-metrics/                # ← you provide this
│   ├── eval/run_metrics.sh
│   └── shape_predictor_68_face_landmarks.dat
└── inputs/                            # Auto-generated by generate_input_files.sh
    ├── hdtf_latentsync.txt
    └── hallo3_latentsync.txt
```

## Setup

### 1. Install conda env

The `omniavatar` conda environment must be available with: torch, transformers, peft,
insightface, onnxruntime-gpu, librosa, opencv-python, etc.

For metrics, the `latentsync-metrics` env is needed (or use the same env if compatible).

### 2. Transfer data

All data files (pretrained_models, checkpoints, validation_data, latentsync-metrics)
should be placed directly under this repo root, matching the layout above.

### 3. Generate input files

```bash
bash scripts/generate_input_files.sh
```

This scans `validation_data/{hdtf,hallo3}/` and creates `inputs/{hdtf,hallo3}_latentsync.txt`
with absolute paths.

### 4. (Optional) Update conda paths

If `torchrun` and `python` are not on your PATH after activating the conda env,
edit the top of `scripts/run_v2v_eval_phase2_1.3B.sh`:

```bash
TORCHRUN="/path/to/conda/envs/omniavatar/bin/torchrun"
METRICS_PYTHON="/path/to/conda/envs/latentsync-metrics/bin/python"
CONDA_METRICS="/path/to/conda/envs/latentsync-metrics/bin"
```

## Usage

```bash
# Activate conda env first
conda activate omniavatar

# Run with default step (2500)
bash scripts/run_v2v_eval_phase2_1.3B.sh

# Run with specific step
bash scripts/run_v2v_eval_phase2_1.3B.sh 10000
```

Uses GPU 2 (hdtf) and GPU 3 (hallo3) by default. Edit the `run_pipeline` calls
at the bottom of the script to change GPU assignments.
